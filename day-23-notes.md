1. What is a branch ?
A branch is an independent line of development in Git.

2. Why use branches?
To develop features without affecting the main branch.

3. What is HEAD?
HEAD is a pointer to the current branch/commit.

4. What happens when switching branches?
Git changes files to match that branch.

5. Origin vs Upstream
Origin = Your GitHub repository.
Upstream = Original repository from which a fork was created.

6. Fetch vs Pull
Fetch downloads changes only.
Pull downloads and merges changes.
